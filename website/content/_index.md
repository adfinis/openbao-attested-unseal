`openbao-attested-unseal` is a design-stage project for OpenBao Auto Unseal
using attested node or workload identity. The project keeps TPM as one backend,
while leaving room for internal-network brokered unseal flows for VM and
Kubernetes deployments.

Operators usually move through [Start Here](/getting-started/),
[Operations](/operations/), [Reference](/reference/), and
[Security](/security/). Maintainers should use [Architecture](/architecture/)
for design rationale and trust boundaries, while contributors should use
[Development](/development/) for local workflow, CI, and release process.
